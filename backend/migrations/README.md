Use Alembic for schema changes.

Recommended commands:

- `backend/.venv/Scripts/python -m alembic upgrade head`
- `backend/.venv/Scripts/python -m alembic revision --autogenerate -m "describe change"`
- Review autogenerated revisions before applying them
- Apply migrations in staging before production
- In production, run with `EVENTCAPTURE_SCHEMA_MANAGEMENT_MODE=validate`

Production startup now validates the schema and will fail fast if required tables are missing.
Do not rely on API startup to create or mutate production tables.

Notes:

- `20260509_0001` is the baseline migration for fresh installs
- `20260510_0002` adds first-customer hardening tables and support workflow fields
